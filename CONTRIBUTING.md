# 贡献指南

感谢你为本仓库贡献内容。

## 贡献范围

你可以通过以下方式参与：
- 新增课程笔记。
- 改进已有内容表述。
- 补充实验材料与可运行代码。
- 增加习题与题解。
- 整理高质量外部资源。
- 修复错别字、失效链接、排版问题。

## 贡献流程

1. Fork 本仓库。
2. 从 `main` 创建分支。
3. 进行聚焦且清晰的修改。
4. 完成自检（格式、链接、结构）。
5. 发起 Pull Request。

## 分支命名建议

建议使用以下格式：
- `文档/<topic>`
- `笔记/<course>-<chapter>`
- `fix/<short-description>`
- `chore/<short-description>`

示例：
- `笔记/数据结构-链表`
- `fix/readme-broken-links`

## Commit 信息建议

推荐使用简洁前缀：
- `docs:`
- `notes:`
- `fix:`
- `chore:`
- `refactor:`

示例：
- `notes: 补充图遍历笔记`
- `fix: 修正死锁定义中的笔误`

## 内容规范

- 课程目录使用中文课程名（如 `课程/数据结构`）。
- 课程内部子目录建议使用固定中文名：`笔记/`、`实验/`、`习题/`、`课件/`、`资源/`。
- 文本内容使用 Markdown（`.md`）。
- 代码示例使用带语言标识的代码块。
- 结论尽量可验证，避免模糊描述。
- 不提交受版权限制且不可公开分发的内容。

## PR 提交前检查

- [ ] 修改范围清晰，便于审阅。
- [ ] 新文件放在正确目录。
- [ ] Markdown 标题层级与链接可正常渲染。
- [ ] PR 描述清楚说明“改了什么、为什么改”。

## 评审约定

- 维护者可能提出修改建议后再合并。
- 讨论聚焦技术与证据，保持尊重。
- 小而聚焦的 PR 通常审阅更快。

## 需要帮助？

可以提交 Issue 并说明：
- 想贡献的内容范围。
- 当前阻塞点或疑问。
- 关联课程或章节。
