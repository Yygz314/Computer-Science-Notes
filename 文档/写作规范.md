# 写作规范

## 命名规则

- 课程目录使用中文课程名（如 `课程/操作系统`）。
- 课程内部子目录使用固定中文名：`笔记/`、`实验/`、`习题/`、`课件/`、`资源/`。
- 名称保持简洁且语义明确。
- 路径应保持统一，不同课程不要混用多种命名方式。

## Markdown 规则

- 每个文件仅保留一个一级标题 `#`。
- 标题层级按顺序使用（`##`、`###`），不跳级。
- 优先使用短段落与列表表达。
- 只有在对比信息时才使用表格。

## 代码片段规范

- 必须使用围栏代码块。
- 尽量标注语言类型。

示例：

````markdown
```cpp
int main() { return 0; }
```
````

## 笔记撰写建议

- 先写定义，再写例子。
- 尽量解释“为什么”，不仅是“是什么”。
- 对关键结论附上参考来源。
- 对未完善部分标记 TODO，便于后续补全。

## 资源记录建议

每条外部资源建议包含：
- 标题
- 链接
- 主题
- 价值说明
